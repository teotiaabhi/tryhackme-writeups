1Ô∏è‚É£ Identity Inventory
Definition

Identity inventory = database of all corporate identities:

Employees (user accounts)

Service accounts

Machine accounts

Contains:

Full name

Role / Job title

Department

Manager

Working hours

Access privileges

Contact details

Why SOC Needs It

Used during alert triage to answer:

Who is this user?

Is this action normal for their role?

Is this within working hours?

Should they have access to this system?

Example Use Case

Alert: G.Baker accessed Finance Server.

Check:

Is G.Baker in Finance?

Does their role require financial file access?

Was access during normal hours?

If yes ‚Üí Possibly normal
If no ‚Üí Suspicious

Sources of Identity Inventory

Active Directory (On-prem AD, Entra ID)

SSO Providers (Okta, Google Workspace)

HR Systems (BambooHR, SAP)

Custom Excel/CSV maintained by IT

2Ô∏è‚É£ Asset Inventory
Definition

Asset inventory = list of all corporate IT assets (servers & workstations).

Contains:

Hostname

IP address

Subnet

Department owner

Purpose of server

Data stored

Installed software

Why SOC Needs It

To understand:

What is this server used for?

Is it sensitive?

Who should access it?

Example

HQ-FINFS-02

Breakdown:

HQ ‚Üí Headquarters

FIN ‚Üí Finance

FS ‚Üí File Server

02 ‚Üí Instance number

Likely stores:

Financial records

Accounting data

If Sales employee accesses it ‚Üí Investigate.

Sources of Asset Inventory

Active Directory

SIEM / EDR (Elastic, CrowdStrike)

MDM (Intune, Jamf)

Custom Excel sheets

3Ô∏è‚É£ Network Diagrams
Definition

Visual map of:

Subnets

Firewall

Servers

VPN

Connections between networks

Why SOC Needs It

To understand:

What subnet an IP belongs to

What services are exposed

Attack path reconstruction

Lateral movement attempts

Example Scenario

External IP: 103.61.240.174
Port: TCP/10443

10443 = VPN service

Steps:

External IP brute forces VPN

Gets internal IP (10.10.0.53)

Scans Database subnet (172.16.15.0/24)

Scans Office subnet (172.16.23.0/24)

Interpretation:

Compromised VPN credentials

Internal reconnaissance

Lateral movement attempt

Goal:

Created by:

Used mostly by:

SOC L1 Analysts

Workbook Structure

3Ô∏è‚É£ Escalation

6Ô∏è‚É£ Email with Script/Binary Attachment Workbook Flow

Correct order:

Analyze email (SPF/DKIM/domain reputation)

Analyze attachment (sandbox/manual review)

Check recipient login activity

Gather triage evidence

Escalate to L2 (if malicious)

Do NOT:

Notify entire company without approval

Skip attachment analysis

Close alert without evidence
